Issue

Every low level task in JIRA is considered an issue. A requirement is an issue. A bug is an issue. A task is an issue. Each issue can be well detailed. You can provide a description; set a priority; attach required screenshots or documents; label the issue for quick access. You can assign the issue to one of your team members, One more important feature is that your team members can have a discussion on the issue in the form of comments.

Issue type:

Story: A user story is the smallest unit of work that needs to be done.

Epics:are large bodies of work that can be broken down into a number of smaller tasks (called stories).

Task: A task represents work that needs to be done.By default, business projects come with one child issue type

Bug: A bug is a problem which impairs or prevents the function of project

Create an issue (Ticket)

{width=“6.03125in” height=“2.8125in”}

Step 1. Click on the create button.

Step 2 select the project

Step 3 Select the issue type that is Story, Epic, Bug, Task

Step 4 Give your Issue Proper Description and add attachment(if any).

Step 6. Add the Asignee (You can choose one from the people present in the project)

Step 7. Add work priority of an issue by clicking on the priority dropdown (Highest, High, Medium, Low, Lowest)

Step 8, Select the Sprint of a project.

Step 9. At last select the Create option.

And your issue or ticket has been created.

SPRINT : A sprint is a short, time-boxed period( 1 to 4 weeks) when a scrum team works to complete a set amount of work

Create a Sprint

Login to your JIRA Scrum Board. Click on the Create sprint button.

Note: Only leaders with specific rights( Project managers, TL, Product Owner, Scrum Master) can create a sprint.
